Capability Map

Davvy is built for day-2 operations, not demos.

The platform combines web administration, permission-aware data models, and a native SabreDAV server so deployment and ongoing maintenance stay in one coherent system.

Start with Docker API Reference

Core surfaces

  • Web app shell: dashboard, contacts, review queue, profile, and admin control center
  • JSON API: role-aware endpoints under /api/*
  • DAV service: CalDAV/CardDAV under /dav/* with autodiscovery redirects

Autodiscovery: /.well-known/caldav and carddav

Both redirect to the main DAV endpoint for client setup.

Identity and Access

One account model, two auth channels

Session-based web auth and DAV basic auth share the same principals, roles, and app settings.

Calendars and Address Books

Default provisioning and exports

New users receive personal defaults, and all collections can be exported individually or in bulk.

Sharing Model

Granular permissions

read_only, editor, and admin sharing roles are enforced across web operations and DAV requests.

Managed Contacts

Sync + moderation workflows

Contact data syncs both directions with vCards and supports review queues for cross-owner edits.

Interoperability

Strict-first payload validation

Calendar and card payloads are normalized and validated, with an admin-controlled compatibility toggle for edge clients.

Operations

Production guardrails

Includes startup preflight checks, auth throttling, and deployment guidance for Railway and Coolify.

Milestone Calendars

Birthday and anniversary generation

Address books can generate and maintain dedicated milestone calendars based on managed contacts.

Apple Compatibility

Optional source mirroring

Selected address books can mirror into a default contacts collection for improved Apple client visibility.

Deploy where you need

Single app, full DAV stack

Davvy runs as one Laravel application serving web UI, API, and DAV. Start with the Docker quick-start, then move to your preferred host.
